# EpubFabric 固定レイアウト開発方針

## 1. 方針転換

EpubFabricは、PDFの内容を章、段落、図版などへ再構成するリフロー型変換を主目的とせず、PDFの紙面を崩さずにEPUB 3へ格納する固定レイアウト変換を主目的とする。

表示の正しさはPDFページを描画した画像によって保証する。PDFの文字レイヤーまたはOCR結果は、表示を組み直すためではなく、検索、選択、コピー、読み上げを補助する透明なテキスト層として使用する。

既存のリフロー型変換処理は比較・実験用として残すが、通常の`convert`と`export`では固定レイアウトを既定とする。

## 2. 成功条件

初期リリースでは次を満たす。

1. PDFの1ページをEPUBの1ページとして出力する。
2. EPUBの表示結果がPDFページ画像と同じであり、OCRやレイアウト解析の成否によって見た目が変化しない。
3. PDFに使用可能な文字レイヤーがあるページでは、その文字列と座標を再利用する。
4. 文字レイヤーがないページでは、OCR文字列と座標を使用する。
5. 透明テキスト層を、可能なEPUBリーダーで検索、選択、コピー、読み上げに利用できる。
6. OCR結果を校正してもページ画像は変更しない。
7. EPUBCheckで重大なエラーがないEPUB 3パッケージを生成する。

全文をリフロー可能にすること、元PDFのフォントを完全に再現すること、複雑な紙面を意味構造へ完全変換することは初期リリースの成功条件に含めない。

## 3. 基本構造

変換経路は次のとおりとする。

```text
PDF
├─ ページを画像化 ───────────────────────┐
│                                        │
├─ 使用可能な文字レイヤーあり             │
│  └─ 文字列と座標を抽出                  │
│                                        ▼
└─ 使用可能な文字レイヤーなし        固定レイアウトXHTML
   └─ OCRで文字列と座標を取得        ├─ ページ画像
                                    └─ 透明テキスト層
                                             │
                                             ▼
                                      EPUB 3パッケージ
```

各XHTMLは次の要素から構成する。

- ページ全体を覆う原型保持用画像
- PDFまたはOCRから得た座標付きテキスト
- ページ寸法を指定する`viewport`
- ページ画像とテキスト層を重ねる固定レイアウトCSS

テキスト層は`display: none`、`visibility: hidden`、`aria-hidden="true"`を使用しない。視覚表示だけを透明にし、DOMとアクセシビリティツリーには残す。

## 4. EPUBパッケージ

`package.opf`には固定レイアウト指定を記録する。

```xml
<meta property="rendition:layout">pre-paginated</meta>
<meta property="rendition:orientation">auto</meta>
<meta property="rendition:spread">auto</meta>
```

spineにはPDFページ順でXHTMLを1件ずつ登録する。初期実装ではPDFのページ順を変更せず、左から右へ進むものとする。右開き、見開き、表紙の単独配置は後続段階で設定可能にする。

```text
book.epub
├── mimetype
├── META-INF
│   └── container.xml
└── EPUB
    ├── package.opf
    ├── nav.xhtml
    ├── styles
    │   └── fixed-layout.css
    ├── text
    │   ├── page-0001.xhtml
    │   └── page-0002.xhtml
    └── images
        ├── page-0001.png
        └── page-0002.png
```

`nav.xhtml`には通常の目次に加えてページリストを生成する。PDFのしおりを取得できるようになった段階で、ページだけの目次から文書目次へ拡張する。

## 5. ページ座標

中間モデルの座標は、従来どおりページ幅・高さに対する0から1の比率で保持する。

XHTML生成時に次の式でページ座標へ変換する。

```text
left   = bounds.x      × viewport.width
top    = bounds.y      × viewport.height
width  = bounds.width  × viewport.width
height = bounds.height × viewport.height
```

ページコンテナには`viewport`と同じ幅・高さをピクセル値で明示する。ページ画像にも同じ`width`と`height`を設定し、`object-fit: contain`で元の縦横比を維持する。リーダーの表示領域に対して幅と高さを別々に引き伸ばしてはならない。

文字列の表示寸法は検索・選択範囲の目安であり、ページ画像の見た目には影響させない。

## 6. 文字情報の選択

### 6.1 PDF文字レイヤー

文字レイヤーから文字列と座標を取得できる場合は、OCRを実行せずPDF文字を使用する。文字化け、座標欠落、極端に少ない文字数などを検出した場合は、使用不能としてOCRへ切り替える。

### 6.2 OCR

画像だけのページは、ページ画像にOCRを実行する。認識結果には文字列、座標、信頼度、読み順を保持する。低信頼度の結果もページ画像には影響しないが、校正対象として記録する。

### 6.3 混在ページ

PDF文字とOCRを同時に使用する場合は、座標の重なりと文字列を比較して重複を除去する。初期実装ではページ単位でPDF文字またはOCRのどちらかを選び、領域単位の併用は後続実装とする。

### 6.4 読み順

DOM上のテキスト順序は、画面上の読み順とできるだけ一致させる。横書き、段組み、縦書き、回転文字について読み順を個別に検証する。柱、ノンブル、装飾文字は必要に応じて読み上げ対象から除外する。

## 7. 原型保持の規則

- ページ画像から本文、図、表などを切り出して再配置しない。
- OCR文字列を可視文字としてページ画像の代わりに描画しない。
- OCR、Ollama、レイアウト解析の結果でページ画像を変更しない。
- 傾き補正やノイズ除去を行う場合も、OCR用画像と表示用画像を分離する。
- 表示用画像の圧縮設定は選択可能にし、既定値で文字が劣化しないようにする。
- 校正は透明テキスト層だけへ反映する。

## 8. 実装単位

### 段階1：固定レイアウトEPUB基盤

- `FixedLayoutEpubPackageBuilder`を追加する。
- 1ページ1XHTMLを生成する。
- ページ画像と透明テキスト層を格納する。
- `convert`と`export`の既定を固定レイアウトにする。
- `--layout reflow`で既存方式を明示的に選択できるようにする。

### 段階2：文字レイヤー品質

- PDF文字レイヤーの使用可否判定を追加する。
- OCRとの重複除去を追加する。
- 行、単語、回転、縦書きの座標精度を改善する。
- 校正結果をテキスト層へ反映する。

### 段階3：表示互換性

- Thorium Reader、Apple Books、Calibreで表示を確認する。
- 各リーダーで検索、選択、コピー、読み上げを確認する。
- 右開き、見開き、表紙単独表示を追加する。
- EPUBCheckを変換後の標準工程へ組み込む。

### 段階4：GUI

- PDF読み込みと変換設定
- ページ画像と透明テキスト層の確認
- OCR文字列の校正
- EPUBプレビュー
- 変換進捗と検証結果

## 9. テスト方法

### 9.1 自動テスト

- `mimetype`がZIP先頭かつ無圧縮である。
- 固定レイアウト用メタデータが存在する。
- PDFページ数とXHTML数、spine項目数、ページ画像数が一致する。
- XHTMLのviewportとページ寸法が一致する。
- ページコンテナと画像の幅・高さがviewportと一致し、画像に`object-fit: contain`が適用される。
- 正規化座標が正しいCSS座標へ変換される。
- PDF文字またはOCR文字がXHTMLへ格納される。
- 除外対象と校正済み文字列が正しく反映される。
- XMLで使用できない文字を除去できる。

### 9.2 画像比較

PDFページとEPUBページを同じ寸法で画像化し、画素差を測定する。透明テキスト層の有無でスクリーンショットが変化しないことを確認する。

### 9.3 実機確認

対象リーダーごとに次を記録する。

| 項目 | 確認内容 |
| --- | --- |
| 表示 | ページ欠落、余白、縦横比、見開き |
| 検索 | PDF文字、OCR文字、日本語、英数字 |
| 選択 | 選択範囲、コピー結果、選択順 |
| 読み上げ | DOM順、除外文字、言語 |
| 拡大 | 画像とテキスト層の位置ずれ |

## 10. CLI契約

```text
epubfabric convert input.pdf --output book.epub
epubfabric convert input.pdf --layout fixed
epubfabric convert input.pdf --layout reflow
epubfabric analyze input.pdf --project book.efproj
epubfabric export book.efproj --format epub --layout fixed
```

`--layout`を省略した場合は`fixed`とする。リフロー方式は実験機能として明示指定時だけ使用する。

## 11. 開発時の確認手順

変更単位ごとに次を実行する。

```powershell
dotnet test EpubFabric.slnx
dotnet run --project src/EpubFabric.Cli -- convert tests/EpubFabric.Tests/Assets/sample-text.pdf --output output/sample-text.epub
git diff --check
```

生成したEPUBはZIP構造、XHTML、OPFを確認し、利用可能な環境ではEPUBCheckと対象リーダーでも確認する。
